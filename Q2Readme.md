# Question 2 – Mathematical Expression Evaluator

## 1. Introduction

This program creates a mathematical expression evaluation system using Python programming language. The system processes expressions that are read from a text file based on a structured parsing method and evaluates the result as per mathematical rules.

The solution employs recursive descent parsing which deals with each operator precedence level in a separate function. The token and parse tree for each expression are generated by the program apart from computing the results. One of the major challenges in this task was correctly handling negative numbers and operator precedence rules, nested parentheses, error conditions of invalid characters, and division by zero.

---

## 2. System Overview

The program processes expressions from an input file (`sample_input.txt`) and generates an output file (`output.txt`). Each expression is analysed and produces four components:

- The original input expression
- A parse tree representation
- A list of tokens
- The computed result

The program strictly follows the output format required in the assignment.

---

## 3. Tokenisation Process

The first stage of the system is tokenisation. This involves scanning each character in the input expression and grouping them into meaningful tokens.

The valid token types include:

| Token | Description |
|-------|-------------|
| `NUM` | Numeric values |
| `OP` | Operators (`+`, `-`, `*`, `/`) |
| `LPAREN` | Opening parenthesis `(` |
| `RPAREN` | Closing parenthesis `)` |
| `END` | End of input |

For example, the expression:

```
-(3 + 4)
```

is tokenised as:

```
[OP:-] [LPAREN:(] [NUM:3] [OP:+] [NUM:4] [RPAREN:)] [END]
```

If an invalid character (e.g., `@`) is encountered, the program immediately terminates processing for that expression and returns an error.

---

## 4. Parsing Method (Recursive Descent)

The program uses recursive descent parsing to analyse expressions according to operator precedence. Four parsing functions are implemented:

### 4.1 `parse_expression()`
Handles the lowest precedence operations:
- Addition (`+`)
- Subtraction (`−`)

### 4.2 `parse_term()`
Handles medium precedence operations:
- Multiplication (`*`)
- Division (`/`)

### 4.3 `parse_unary()`
Handles unary negation, such as:
- `-5`
- `--5`
- `-(3+4)`

> **Note:** Unary plus is not supported and produces an error.

### 4.4 `parse_primary()`
Handles the highest precedence elements:
- Numeric values
- Parenthesised expressions

---

## 5. Operator Precedence Handling

Operator precedence is enforced through the order of function calls:

```
parse_expression → parse_term → parse_unary → parse_primary
```

This ensures:
- Multiplication and division are evaluated before addition and subtraction
- Parentheses are evaluated first
- Unary negation is applied correctly

For example:

```
2 + 3 * 4
```

is evaluated as:

```
(+ 2 (* 3 4))
```

which produces the correct result: **14**

---

## 6. Parse Tree Representation

The program constructs a parse tree to represent each expression. The required formats are:

| Type | Format |
|------|--------|
| Number | `5` |
| Binary operation | `(+ 3 5)` |
| Unary negation | `(neg 5)` |

**Example:**

Input:
```
(10 - 2) * 3 + -4 / 2
```

Tree representation:
```
(+ (* (- 10 2) 3) (/ (neg 4) 2))
```

---

## 7. Evaluation Process

After constructing the parse tree, the program evaluates it recursively. The evaluation rules are:

- Number nodes return their value
- Unary negation reverses the sign
- Binary operations compute left and right values

**Examples:**

```
(neg (+ 3 4))  →  -7
(+ 2 (* 3 4))  →  14
```

---

## 8. Error Handling

The system includes robust error handling to manage invalid inputs.

### 8.1 Invalid Characters

If an invalid character is detected:

```
3 @ 5
```

Output:
```
Tree: ERROR
Tokens: ERROR
Result: ERROR
```

### 8.2 Syntax Errors

If an expression has incorrect structure (e.g., missing parentheses):
- Tokens may still be generated
- Parse tree becomes `ERROR`
- Result becomes `ERROR`

### 8.3 Division by Zero

For expressions such as `1 / 0`:
- Tree is generated correctly
- Tokens are displayed
- Result is: `ERROR`

---

## 9. Output Format

Each expression produces exactly four lines:

```
Input: ...
Tree: ...
Tokens: ...
Result: ...
```

A blank line separates each expression.

**Formatting requirements:**
- Tokens must match exactly
- Tree structure must match exactly
- Whole numbers must not include decimals (e.g., `8` not `8.0`)
- Decimal values must be rounded to 4 decimal places
